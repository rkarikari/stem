Assembly Steps:

  1. Power System:
    • Mount solar panel on tilting frame (optimized for latitude)
    • Install charge controller in weatherproof enclosure
    • Connect battery with appropriate fusing (30A)
    • Wire Anderson Powerpole distribution panel
  2. Radio Installation:
    • Mount transceiver in equipment box
    • Connect 12V power with inline fuse
    • Install antenna on 6m mast with guy wires
    • Test SWR on both 2m and 70cm bands
  3. Monitoring System:
    • Raspberry Pi with voltage/current sensors
    • Real-time power dashboard via web interface
    • Data logging to SD card (CSV format)
    • Solar panel angle optimization display
Energy Flow & Runtime Calculation
💡 Pro Tip: Actual field operations alternate between transmit and receive (typical 20% TX duty cycle), extending runtime to 8+ hours. Monitor battery SOC and avoid discharge below 30%.

Testing Protocol:

  1. Verify solar charging in full sun (>5A current)
  2. Confirm battery voltage monitoring accuracy
  3. Test radio TX/RX on local repeaters
  4. Measure power consumption during transmit
  5. Calculate runtime: 35Ah × 12.8V / 90W avg = 5 hours TX
  6. Conduct 8-hour field test with mixed TX/RX

Success Metrics:

  • ✅ 8+ hours continuous operation
  • ✅ Battery never below 30% SOC
  • ✅ Successful contacts on 2m and 70cm
  • ✅ All monitoring data logged correctly
  • ✅ Weather-resistant setup (tested in rain)

Project 2: Automated Satellite Tracking System

🛰️ Computer-Controlled Ground Station

Objective: Build station that automatically tracks satellites and corrects Doppler shift

Satellite Ground Station System Architecture
Connections:
Power/Control

RF Signal

Hardware Requirements:

  • Dual-band VHF/UHF transceiver with CAT control
  • Az/El rotator system (Yaesu G-5500 or equivalent)
  • Crossed-Yagi antennas (2m and 70cm)
  • Computer (Raspberry Pi 4 or laptop)
  • GPS module for accurate time
  • Rotator interface (USB or serial)
Doppler Shift Correction Process
📊 Note: VHF satellites (145 MHz) experience ±3 kHz Doppler shift. UHF (435 MHz) sees ±9 kHz shift. Continuous correction ensures clear reception throughout the pass.

Software Components:

  1. Tracking Core: Python + Skyfield for orbital calculations, TLE auto-update from Celestrak, Real-time Az/El computation
  2. Doppler Correction: Calculate frequency shift every 1 second, Send CAT commands to radio, Separate uplink and downlink correction
  3. Rotator Control: Send position commands via serial, Monitor actual antenna position, Smooth tracking (avoid hunting)
  4. User Interface: Web-based dashboard (Flask/HTML), Real-time satellite position display, Pass prediction table, Manual override controls

Key Features to Implement:

  • Automatic pass scheduling (start tracking 2 min before AOS)
  • Multi-satellite database (AO-91, AO-92, SO-50, ISS)
  • Audio recording of each pass
  • Contact logging with grid square extraction
  • Signal strength vs elevation plotting

Calibration Process:

  1. Align antennas to true north using compass
  2. Calibrate elevation zero (horizontal)
  3. Test tracking on Moon (easy, slow-moving target)
  4. Verify Doppler correction on known beacon
  5. Fine-tune timing using GPS reference

Objective: Use drone to extend VHF/UHF range via elevated antenna platform

Drone RF Relay - Coverage Comparison
📡 Coverage Advantage: Elevating antenna to 100m provides line-of-sight over terrain obstacles, extending range from 10km to 50km radius. Perfect for emergency communications and field exercises.

Operating Modes:

  1. Manual Relay: Ground operator keys drone radio via RC channel
  2. Cross-Band Repeat: Receive on 2m, transmit on 70cm automatically
  3. APRS Beacon: Transmit position every 30 seconds
  4. Signal Mapping: Record RSSI at multiple locations

Mission Profile Example:

  • Position: Midpoint between two ground stations

Project 4: AI-Powered Propagation Predictor

🤖 Machine Learning for HF Forecasting

Objective: Train neural network to predict optimal HF operating times

AI Training & Prediction Workflow

Data Collection (3+ months):

  • Time/date of each contact attempt
  • Frequency and band used
  • Solar flux index (daily)
  • K-index (geomagnetic activity)
  • Contact success (yes/no)
  • Signal report if successful (S-units)
  • Distance to station contacted

Model Development:

  • Input features: [hour, day_of_year, solar_flux, k_index, frequency, distance]
  • Output: Probability of successful contact (0-1)
  • Architecture: 3-layer neural network or Random Forest
  • Training: 80% data for training, 20% for validation

Deployment:

  • Web dashboard showing "best times" for next 7 days
  • SMS/email alerts when conditions are optimal
  • Comparison to traditional propagation tools
  • Continuous learning from new contacts
