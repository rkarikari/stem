RadioSport students don't just learn technology‚Äîthey build production-grade applications that solve real problems in amateur radio and aerospace. üéØ The Capstone Philosophy

By Month 8-9 of the RadioSport program, students have acquired deep technical knowledge across radio, satellites, drones, AI, and software development. The capstone project is where they synthesize this knowledge into a functional, deployed application that provides genuine value to society.

Key Requirements:

  • Must solve a real problem (not just a demo)
  • Publicly accessible via download or web URL
  • Production-ready code quality
  • User documentation included
  • Demonstrates multi-domain integration

Exemplar Capstone Projects

Three Development Platforms

  1. Android Mobile Apps: Qt/C++ applications for handheld amateur radio operations
  2. Windows Desktop Apps: Qt/C++ applications for SDR and signal processing
  3. Web Applications: Python/Streamlit or HTML/JavaScript tools for cloud-based operations

Students typically work in teams of 2-4, with each member responsible for specific features while ensuring overall integration. Projects are judged on technical merit, real-world utility, code quality, and innovation.

Case Study: Building SatTrackRadio

From Concept to Production: A 12-Week Journey

Problem: Amateur radio satellite operators need real-time Doppler frequency correction and satellite tracking, but existing solutions are either PC-only, require expensive commercial radios with built-in correction, or lack integration with handheld transceivers.

Solution: Build an Android application that combines SGP4 orbital propagation, real-time Doppler calculations, and direct Bluetooth CAT control for affordable handheld transceivers (TH-D74/D75, FT-818).

Technical Approach:

  1. Research Phase (Week 1-2):
    • Study SGP4 propagation algorithm and orbital mechanics
    • Reverse-engineer Kenwood TH-D74 CAT protocol (ASCII commands)
    • Learn Yaesu FT-818 CAT protocol (binary commands, BCD encoding)
    • Evaluate Android GPS and Bluetooth APIs
    • Research TLE data sources (Celestrak) and transponder databases (SatNOGS)
  2. Core Development (Week 3-6):
    • Implement SGP4 propagator with TLE parser
    • Build Bluetooth serial communication layer
    • Create CAT command interface for both radio types
    • Develop real-time satellite position calculator
    • Implement Doppler shift formulas for uplink/downlink
  3. Advanced Features (Week 7-10):
    • GPS integration with time synchronization
    • Pass prediction algorithm with configurable elevation
    • Automatic frequency updating during satellite passes
    • TLE auto-download and update system
    • Transponder database integration
    • Multi-mode support (RX-only, TX-only, Full Duplex)
  4. Polish & Release (Week 11-12):
    • Field testing with actual satellite passes
    • UI refinement and user experience optimization
    • Comprehensive user guide with screenshots
    • Package as Android APK
    • GitHub release with documentation
    • Community announcement on amateur radio forums
// Doppler Frequency Correction Algorithm // Calculate Doppler shift based on satellite velocity and observer position double calculateDopplerShift(double baseFreq, Vector3d satVelocity, Vector3d satPosition, Vector3d obsPosition) { // Calculate range vector from observer to satellite Vector3d rangeVec = satPosition - obsPosition; double range = rangeVec.norm(); // Calculate range rate (velocity along line of sight) double rangeRate = rangeVec.dot(satVelocity) / range; // Doppler shift formula: Œîf = -f‚ÇÄ √ó (v/c) // Negative because approaching satellite increases frequency double speedOfLight = 299792.458; // km/s double dopplerShift = -baseFreq * (rangeRate / speedOfLight); return dopplerShift; } // Apply to uplink and downlink frequencies double uplinkFreq = 145.950; // MHz double downlinkFreq = 435.750; // MHz // Uplink: Pre-compensate for satellite receiving Doppler-shifted signal double correctedUplink = uplinkFreq - calculateDopplerShift(uplinkFreq, satVel, satPos, obsPos); // Downlink: Compensate for Doppler shift in received signal double correctedDownlink = downlinkFreq + calculateDopplerShift(downlinkFreq, satVel, satPos, obsPos); // Send frequency commands to radio via Bluetooth CAT sendCAT_SetFrequency(BAND_A, correctedUplink); sendCAT_SetFrequency(BAND_B, correctedDownlink);
